=============================================================================
TECHNICAL DETAILS
=============================================================================

The changes modify only constant definitions. All existing code references
these constants (MON_TABLE_ENTRIES, MOVE_TABLE_ENTRIES, etc.) so the 
changes automatically apply throughout the codebase without requiring
code modifications.

Files verified to use constants properly:
  - engine/pokemon/correct_party_errors.asm
  - engine/pokemon/mon_menu.asm
  - engine/overworld/scripting.asm
  - home/pokemon.asm

RAM allocation (wPokedexOrder) was already sized for 1407 Pokemon, so
no memory changes were needed.

The UI's PrintNum function supports up to 7 digits, so displaying the
Pokemon count is not an issue.

=============================================================================
PERFORMANCE IMPROVEMENTS
=============================================================================

Larger cache sizes and increased saved recent indexes will improve
performance by reducing:
  - Table evictions
  - Cache misses
  - Memory reallocations during gameplay

KNOWN ISSUES
=============================================================================

BUILD ISSUE: Many newer Pokemon sprites are in RGBA format instead of
indexed color with embedded palettes. This prevents building until fixed.

Affected sprites include: arceus, sylveon, togekiss, azurill, porygonz,
and many others from the ToDo list.

SOLUTION: See FIXING_SPRITES.md for detailed fix instructions.
A script is provided to automatically convert all RGBA sprites.
